# Weight-1 vs weight-2 Pauli training-set comparison

A/B test on `Learning_TDME_scheduler`:
do the extra training samples in the weight-2 Pauli basis help the PQC learn the same target Lindbladian?

## Setup

- N = 8 qubits, T_model = 3 layers, target TDME at L_target ∈ {3, 4, 5, 6, 7}.
- Fixed (μ=1, γ=0.1, J=1, t=1.0), on-site Z-dephasing.
- Training sets:
  - **weight-1**: all single-site Paulis on N sites → 3N = 24 inputs.
  - **weight-2**: all (X_iX_j, Y_iY_j, Z_iZ_j) for i<j → 3N(N−1)/2 = 84 inputs.
- Epochs chosen so that total per-sample evaluations are roughly equal:
  - weight-1: 200 + 40 fine-tune = 240 → 24 × 240 ≈ 5760 evals
  - weight-2: 60 + 12 fine-tune = 72 → 84 × 72 ≈ 6048 evals
- Same target Lindbladian (no random `param`), so both weights chase identical teachers per L_target.

## Files

- `run_weight1_vs_weight2.py` — runner; spawns 10 workers (one per (L, weight))
  inside a single Trillium node.
- `job_weight1_vs_weight2.sh` — sbatch wrapper, 1h debug-partition walltime.
- `plot_weight1_vs_weight2.py` — overlay learning-loss curves and dump summary CSV.
- `results/` — per-task output (`*_learning_loss.npy`, `*_testing_loss.npy`,
  `*_testing_loss_list.npy`), naming pattern
  `N8_T3_L{L}_t1.0_g0.1_w{1|2}_{kind}.npy`.
- `figures/` — generated by `plot_weight1_vs_weight2.py`.

## Run

```bash
cd /scratch/simba/Learning_Lindbladian/weight1_vs_weight2
sbatch --partition=debug job_weight1_vs_weight2.sh
# then, once complete:
python plot_weight1_vs_weight2.py
```
